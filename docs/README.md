# 双色球分析标注

当前确认版本：正式版 2.1。

正式版入口只有两个：

- Windows：双击根目录的 `打开双色球分析.vbs`。
- macOS：双击根目录的 `打开双色球分析.app`。

两个入口都会启动本地服务，并自动打开浏览器。浏览器不能通过 HTML 文件直接启动本地程序，因此不再提供通用 HTML 启动文件。

## 目录

- `../app/`：页面入口与标注、预测、策略、回测界面。
- `../app/core/`：跨页面共享的存储、号码处理与标注几何模块。
- `../data/`：本地开奖数据快照与历史回测预测数据。
- `../scripts/`：本地服务与数据刷新脚本。
- `./`：项目说明文档。

## 使用

1. Windows 双击根目录的 `打开双色球分析.vbs`；macOS 双击根目录的 `打开双色球分析.app`。
2. 浏览器会在本地服务启动后自动打开页面；端口默认使用 8765，已有服务时会复用。
3. 点击“刷新图表”或顶部刷新按钮，可通过本地服务更新开奖数据。

标注记录会自动保存在浏览器本地；使用本地服务打开时，也会同步保存为 `../data/ssq-analysis-records.json`，该文件不纳入版本管理。

## 启动与故障排查

- 启动页会验证 `8765` 端口上的服务是否确实为本项目；若提示端口被占用，请关闭占用该端口的程序后重新打开入口。
- Windows 使用系统自带 PowerShell 启动服务；macOS 需要已安装并可在终端中使用 Python 3。
- 本地服务只公开页面和必要的数据文件，不会提供项目脚本、文档或隐藏文件。
- 记录文件单次最大 2 MB；保存内容未变化时不会重复生成备份。服务最多保留 30 份历史备份。

## 数据说明

- 图表刷新从 500彩票网读取最近开奖数据；网络异常或源页面结构变化时，应用会保留现有本地数据。
- 刷新后服务会校验期号顺序、红球数量和范围、蓝球范围以及最新期号一致性；校验失败不会写入新的数据文件。

## 开发约定

- 跨页面的号码解析、预测记录读写、期号处理统一放在 `app/core/ssq-core.js`，页面不得再次复制同类工具函数。
- 标注的边界、命中和缩放几何统一放在 `app/core/annotation-geometry.js`。
- 页面只负责界面与业务编排；跨页面状态继续使用同一份记录 schema，修改 schema 时同步更新核心模块和版本号。
- 新增浏览器资源需同时加入 Python 与 PowerShell 本地服务的公开路径白名单，避免 Windows 与 macOS 行为不一致。
